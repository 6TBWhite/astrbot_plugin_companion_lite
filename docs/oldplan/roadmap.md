# CompanionLite 重构路线图

> 2026-07-08 更新：本路线图描述的是完整愿景。实际执行以 `execution-plan-slim.md`（瘦身执行计划）为准：
> 阶段 2/2.5 已冻结（不再新增字段、事件、模板），阶段 3/4 按瘦身版实现，阶段 5 无限期搁置。

## 一、最终愿景

CompanionLite 的最终目标不是做一个事实记忆插件，也不是替代回复生成插件，而是成为 AstrBot 私聊陪伴场景中的关系连续性层。

它负责维护“我和这个用户如何相处”：当前关系阶段、边界压力、互动能量、表达偏好、每日情感弧线，以及最近几天的相处趋势。事实记忆继续交给 LivingMemory 管理，CompanionLite 只通过只读方式使用 LivingMemory 的近期摘要来增强连续性，不主动写入 LivingMemory。

最终效果是：LLM 在回复前不仅知道用户过去说过什么，也知道昨天/今天的相处状态如何、最近关系是在靠近还是疲惫、此刻应该更主动还是更收敛。

核心定义：

- LivingMemory 管“发生了什么”。
- CompanionLite 管“我们现在如何相处”。
- LLM 负责融合事实记忆和关系连续性，生成自然回复。
- CompanionLite 不抢答、不做人设自动进化、不重复建设事实记忆系统。

## 二、设计边界

必须坚持的边界：

- 不自动绑定用户，只处理后台明确配置的 UID。
- 不默认学习群聊。
- 不主动写入 LivingMemory。
- 不大量搬运 LivingMemory 原文进入自己的上下文。
- 不把内部数值直接暴露给用户或 LLM。
- 不为了陪伴感牺牲边界感。
- 不让 LivingMemory 成为硬依赖，读不到时必须安全降级。

推荐的职责分层：

- `BindingManager`：读取后台配置的绑定 UID，决定是否处理当前用户。
- `EventEngine`：基于规则识别即时互动事件。
- `StateEngine`：维护即时关系状态、能量恢复、边界恢复和时间衰减。
- `ArcEngine`：生成每日情感弧线。
- `ContinuityEngine`：根据最近几天弧线生成加权连续性提示。
- `LivingMemoryReader`：可选只读读取 LivingMemory 近期摘要。
- `ReflectionEngine`：低频 LLM 反思，更新状态、弧线和连续性。
- `ContextBuilder`：构建最终注入 LLM 的自然语言相处提示。
- `DebugPanel`：展示绑定、状态、弧线、趋势、反思和 LivingMemory 读取状态。

## 三、目标数据模型

### 1. 即时关系状态

`CompanionState` 表示此刻如何回复。

主要字段：

- `familiarity`：熟悉度。
- `closeness`：亲近度。
- `safety`：安全感。
- `boundary_pressure`：边界压力。
- `energy`：互动能量。
- `mood`：bot 当前相处状态，不表示用户真实情绪。
- `last_event`：最近一次识别事件。
- `last_deep_reflection_at`：最近一次深度反思时间。
- `last_reflection_summary`：最近一次反思摘要。

### 2. 每日情感弧线

`DailyArc` 表示某一天的整体相处走势。

建议字段：

- `user_id`：绑定用户 UID。
- `date`：日期。
- `overall_mood`：当天整体情绪走势。
- `relationship_trend`：关系趋势，例如靠近、稳定、疲惫、拉扯、恢复。
- `energy_curve`：一天中能量变化。
- `boundary_curve`：边界压力变化。
- `important_interactions`：重要互动摘要。
- `style_observations`：当天观察到的表达偏好。
- `tomorrow_guidance`：明天相处建议。
- `source`：数据来源，例如 local、livingmemory、mixed。
- `updated_at`：更新时间。

示例：

```json
{
  "user_id": "123456",
  "date": "2026-07-07",
  "overall_mood": "低落后缓和",
  "relationship_trend": "轻微靠近",
  "energy_curve": "下午较活跃，晚上明显疲惫",
  "boundary_curve": "晚上有轻微边界压力",
  "important_interactions": [
    "用户下午主动分享工作压力",
    "晚上要求回复短一点",
    "最后对话以疲惫但平和结束"
  ],
  "style_observations": [
    "今天更偏好短回复",
    "不希望连续追问"
  ],
  "tomorrow_guidance": "明天开场保持轻柔，不要太主动追问，优先承接情绪。"
}
```

### 3. 跨日连续性摘要

`ContinuitySummary` 表示最近几天如何影响今天/明天的回复姿态。

建议字段：

- `user_id`：绑定用户 UID。
- `date`：摘要生成日期。
- `recent_emotional_direction`：最近情绪方向。
- `relationship_momentum`：关系动量。
- `risk_flags`：风险提示，例如连续低能量、反复要求少追问。
- `recommended_posture`：建议相处姿态。
- `carry_over_notes`：需要延续到今天/明天的提示。
- `source_days`：使用了最近几天的弧线。
- `updated_at`：更新时间。

建议权重：

- 今天：1.00。
- 昨天：0.75。
- 前天：0.55。
- 3-7 天：0.25 到 0.40。
- 7 天前：只作为长期底色，不直接强烈影响当天语气。

实际注入可按这个比例融合：

```text
当前回复姿态 =
  50% 即时关系状态
+ 30% 今天/昨天情感弧线
+ 15% 近 3 天趋势
+ 5% 长期关系底色
```

### 4. 知识收获记录

CompanionLite 可以让 bot “真材实料地学点什么”，但不应实现成无限制的自由笔记工具。更安全的方式是新增一条受控的 `KnowledgeHarvest` 流水线：bot 可以提出“我似乎学到了一个新东西”，但必须经过结构化、去重、置信度判断和可选审查后，才进入长期可用层。

这个能力不替代 LivingMemory。LivingMemory 仍然保存“用户说过什么”，`KnowledgeHarvest` 保存的是“bot 从对话中学到的可复用知识、概念解释、用户教过的规则、长期相处经验”。

建议字段：

- `user_id`：来源绑定 UID。
- `topic`：主题，例如某个工具、概念、项目、用户自定义规则。
- `claim`：学到的内容，必须是短句或结构化结论。
- `evidence`：来自哪段对话的简短证据摘要，不保存大段原文。
- `confidence`：置信度。
- `scope`：适用范围，例如 user_only、general、project、relationship。
- `source`：local、livingmemory、manual、mixed。
- `status`：pending、approved、rejected、superseded。
- `created_at` / `updated_at`：创建和更新时间。

示例：

```json
{
  "user_id": "123456",
  "topic": "用户的开发习惯",
  "claim": "用户更喜欢先讨论最终愿景，再分阶段实现。",
  "evidence": "用户多次要求先对齐愿景和阶段计划。",
  "confidence": 0.82,
  "scope": "user_only",
  "source": "local",
  "status": "approved"
}
```

必须避免的形式：

- 不让 LLM 任意写自由文本笔记。
- 不让 bot 自己决定所有内容都永久有效。
- 不保存未经压缩的大段私聊原文。
- 不把事实、关系状态、人格改写混在同一张表里。
- 不把低置信度猜测直接注入后续对话。

推荐流程：

```text
对话片段 / LivingMemory 近期摘要
  -> 候选知识提取
  -> 去重与冲突检查
  -> 置信度评分
  -> 可选人工审查
  -> approved 后进入可检索知识层
  -> LLM 请求前按主题少量注入
```

注入时应只注入和当前问题相关的少量知识，不做全量背景灌入。

## 四、阶段 1：稳定地基

### 目标

把现有插件从概念 alpha 调整为稳定、可配置、不会误学习的版本。

### 主要工作

1. 去掉自动绑定逻辑。

- 删除或停用 `_auto_main_user_id`。
- 只处理 `main_user_ids` 中明确配置的 UID。
- 未配置 UID 时不捕获、不注入、不反思。
- Debug 页面显示“未绑定 UID”。

2. 修复配置读取。

- 兼容 AstrBot 配置面板可能传入的分组结构。
- 同时保留当前扁平结构兼容能力。
- 确保 `main_user_ids` 支持逗号字符串和列表两种格式。

3. 修复消息缓冲管理。

- 增加每用户缓冲上限，例如 100 或 200 条。
- 关闭深度反思时也不允许无限增长。
- 反思失败时不清空缓冲。
- 反思成功后再清空或裁剪已处理消息。

4. 增加反思任务去重。

- 同一用户同时只能有一个反思任务。
- 手动触发和自动触发共用同一套任务管理。

5. 修复活跃聊天识别。

- 基于最近 60 秒用户消息数计算 `recent_rate`。
- 让 `active_chat` 能真实触发能量消耗。

6. 清理 LivingMemory 职责混淆。

- 移除或停用 `memory_worthy` / `memory_content`。
- 反思输出只关注关系、情绪弧线、表达偏好和相处建议。

### 验收标准

- 未配置 UID 时插件不会学习或注入。
- 配置 UID 后只处理这些 UID。
- WebUI 配置修改能实际生效。
- LLM 反思失败不会丢失消息缓冲。
- 消息缓冲不会无限增长。
- 同一用户不会并发跑多个反思任务。
- 高频私聊会触发能量下降。

## 五、阶段 2：关系状态重构

### 目标

让关系状态具备恢复、衰减和更自然的边界表达，而不是简单数值累加。

### 主要工作

1. 明确状态语义。

- `mood` 定义为 bot 当前相处状态，不代表用户真实心情。
- 文档、Debug 页面和上下文注入统一这个说法。

2. 增加恢复类事件。

- `apology`：用户道歉或缓和关系。
- `repair`：用户主动解释、重新邀请聊天。
- `comfort`：用户安慰 bot 或表达理解。
- `positive_closure`：对话自然结束且氛围良好。

3. 增加时间恢复/衰减。

- 能量随时间恢复。
- 边界压力随时间和正向互动下降。
- 亲近度长期不聊时轻微冷却，但不剧烈下降。
- 熟悉度作为长期底色，不轻易下降。

4. 调整沉默机制。

- 从“冷淡疏离”改为“收敛、少追问、降低主动性”。
- 保留三档：轻度收敛、防御收敛、休息结束。
- 默认软模式只注入意图，不强行拦截回复。
- 硬模式作为后续可选项，不在本阶段强依赖。

5. 调整状态注入文案。

- 不再注入强烈人格描述。
- 用自然相处建议表达，例如“少追问”“短一点”“先承接情绪”。

### 验收标准

- 高频聊天会降低能量，一段时间后能量会恢复。
- 用户道歉、感谢、缓和互动能降低边界压力。
- 一次负面事件不会永久让 bot 处于防御状态。
- 沉默提示更自然，不会突然变得生硬冷淡。
- Debug 页面能解释当前相处姿态来自哪些事件。

## 六、阶段 3：每日情感弧线与连续性

### 目标

实现“完整的一天的情感弧线”，并让它能指导明天和后天的回复姿态。

### 主要工作

1. 新增 `daily_arc` 表。

- 每个绑定 UID 每天一条。
- 数据来源优先使用 CompanionLite 自己的消息缓冲和事件流。
- 支持被多次更新，当天反思可逐步完善弧线。

2. 扩展 LLM 反思输出。

- 除了状态 delta，还输出当天情感弧线。
- 输出最近一次反思摘要。
- 输出明日相处建议。
- 输出表达偏好观察，但只在用户明确表达偏好时更新长期偏好。

3. 新增 `continuity_summary` 表。

- 根据最近几天 `daily_arc` 加权生成。
- 越近权重越高。
- 输出风险提示、关系动量、推荐姿态和延续提示。

4. 升级上下文注入。

注入内容从“当前数值描述”升级为：

- 当前关系状态。
- 当前相处姿态。
- 今日情感弧线。
- 昨天/近几天延续提示。
- 最近表达偏好。
- 回复建议。

示例：

```xml
<companion_context>
当前关系：比较熟，有稳定信任感。
当前相处姿态：温和、少追问、不要过度活跃。
今日情绪弧线：用户白天压力较高，晚上能量偏低，但整体愿意交流。
连续性提示：昨天最后对话以疲惫结束，今天回复时先轻柔承接，不要突然开启复杂话题。
表达偏好：最近更偏好短回复和直接表达。
</companion_context>
```

5. 升级 Debug 页面。

- 展示今日情感弧线。
- 展示近几天连续性摘要。
- 展示最近一次反思摘要。
- 展示数据来源和更新时间。

### 验收标准

- 每天能生成并更新一条 `DailyArc`。
- 第二天上下文能带上昨天的延续建议。
- 最近几天趋势会影响回复姿态。
- Debug 页面能看到今日弧线、近几天趋势和反思摘要。
- 插件仍然不保存事实记忆，只保存关系和相处连续性摘要。

## 七、阶段 4：LivingMemory 只读增强

### 目标

按保守增强模式读取 LivingMemory：能读就增强每日弧线，读不到就安全降级。

### 主要工作

1. 新增 `LivingMemoryReader`。

- 检测 LivingMemory 是否存在和激活。
- 探测可用只读接口。
- 读取失败不抛出到主流程。
- 记录最近读取状态。

2. 限制读取范围。

- 只读取绑定 UID 相关内容。
- 默认读取最近 1-3 天。
- 限制返回数量和字符数。
- 优先读取摘要，不读取大段原始记录。

3. 作为弧线输入，而不是直接注入。

- LivingMemory 内容只作为 `DailyArc` 构建输入。
- CompanionLite 只提炼情绪走势、关系趋势和相处建议。
- 最终 LLM 注入仍然是 CompanionLite 的关系连续性提示。

4. 增加降级和可观测性。

- Debug 页面显示 LM 是否可读。
- 显示最近读取时间、读取结果数量、是否降级。
- LM 不可用时插件继续依靠本地消息缓冲工作。

### 验收标准

- LivingMemory 可用时，近期摘要能参与 `DailyArc` 构建。
- LivingMemory 不可用或接口变化时，CompanionLite 正常工作。
- 插件不写入 LivingMemory。
- 注入内容不重复搬运事实，只输出关系/情绪连续性建议。
- Debug 页面能明确显示 LM 读取状态。

## 八、阶段 5：知识收获与受控自学习

### 目标

让 bot 能从长期私聊中学到可复用知识，而不是只学习关系状态和表达偏好。这个阶段实现的是受控自学习，不是模型微调，也不是自由笔记。

### 主要工作

1. 新增 `knowledge_harvest` 表。

- 保存结构化知识收获。
- 每条记录必须有 topic、claim、evidence、confidence、scope、status。
- 默认 status 为 pending 或 approved，由配置决定。

2. 新增候选知识提取器。

- 从本地消息缓冲和 LivingMemory 近期摘要中提取候选知识。
- 只提取明确被用户教授、反复确认或对后续有帮助的内容。
- 区分用户私有事实、通用知识、项目知识和相处经验。

3. 新增去重和冲突检查。

- 相同 topic 下合并重复 claim。
- 如果新旧 claim 冲突，旧记录标记为 superseded 或新记录进入 pending。
- 低置信度内容不进入注入层。

4. 新增审查策略。

- 默认可以先使用半自动模式：高置信度自动批准，低置信度 pending。
- Debug 页面展示 pending 知识。
- 管理员可批准、拒绝、删除或改写。

5. 新增相关性检索与注入。

- LLM 请求前根据当前 prompt 检索少量 approved 知识。
- 只注入 claim 和必要 scope，不注入大段证据。
- 知识注入和关系连续性注入分开，避免职责混乱。

### 验收标准

- bot 能把用户明确教过的内容沉淀成结构化知识。
- 低质量、猜测性内容不会自动长期生效。
- 已批准知识能在相关问题中被重新使用。
- Debug 页面能查看、批准、拒绝和删除知识收获。
- LivingMemory 不可用时仍能依赖本地消息缓冲生成候选。
- 插件仍不写入 LivingMemory。

## 九、执行顺序

建议严格按阶段推进：

1. 阶段 1：稳定地基。
2. 阶段 2：关系状态重构。
3. 阶段 3：每日情感弧线与连续性。
4. 阶段 4：LivingMemory 只读增强。
5. 阶段 5：知识收获与受控自学习。

不建议先做 LivingMemory 深度接入。每日情感弧线应是 CompanionLite 自己的核心能力，LivingMemory 只能增强它，不能成为它的前提。

## 十、阶段完成后的最终形态

完成四个阶段后，CompanionLite 应具备以下能力：

- 后台明确绑定 UID，避免误学习。
- 私聊中维护稳定的即时关系状态。
- 能识别靠近、拒绝、疲惫、道歉、缓和、风格偏好等事件。
- 能让能量和边界压力随时间自然恢复。
- 能生成每日情感弧线。
- 能将昨天、前天和近几天的情绪走势加权带入今天回复。
- 能只读 LivingMemory 的近期摘要来补全情感弧线。
- 能从用户明确教授或反复确认的内容中沉淀受控知识。
- 能在 LLM 请求前注入自然语言相处提示。
- 能在相关问题中注入少量已批准知识。
- 能通过 Debug 面板解释为什么 bot 当前应该更主动、更温柔、更短，或更收敛。

最终目标是让 bot 在长期私聊里表现出连续、稳定、有边界的陪伴感：它既记得事实，也记得昨天我们是怎么结束的，以及今天该如何继续。
